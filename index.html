Git Features
  • ahead-behind algorithm and ref-filter token

    The git for-each-ref command now understands %(ahead-behind:base) tokens to output counts for comparing commit histories. This algorithm powers GitHub's branches page.

  • index.skipHash config option

    By enabling this config option, your index writes will speed up dramatically while only losing a small probability of error detection. This will speed up `git status`, `git add`, and `git commit` commands, among others. A version of this feature was previously available in the `microsoft/git` fork and its version of Scalar. Git v2.40.0

  • Bundle URIs

    Bundle URIs provide a way to offload object downloads to static content servers, speeding up clones and fetches while reducing load on the origin Git server. Git v2.38.0 (still under development)

    • Git clients can fetch from bundle lists when using creationToken heuristic
    • Git servers can advertise bundle lists over protocol v2 using the `bundle-uri` command
    • Bundle lists allow a static server to advertise multiple bundles
    • `git clone --bundle-uri`
    • Preparations for bundle URIs & design document

  • git rebase --update-refs option

    A new option allowing a stack of topics to automatically have their tracking topic branches be updated during a rebase. Git v2.38.0

  • Partial bundles

    Git's bundle format pairs refs with a pack-file of objects. It can now include a `filter` capability, allowing a bundle to bootstrap a blobless partial clone. Git v2.36.0

  • The sparse index

    The sparse index reduces the index size when using cone-mode sparse-checkout patterns, creating significant performance boosts for monorepos. Git v2.32.0, v2.33.0, and v2.34.0

    • Sparse index format
    • Protections against hitting sparse directories when not expecting them
    • Integration with git status
    • Integration with git commit and git checkout
    • Integration with git add
    • Sparse-checkout now deletes ignored files outside of the cone
    • Integration with git merge, git rebase, git cherry-pick, and git revert
    • Restrict git add, git rm, and git mv to disallow changing things outside of the sparse cone, unless --sparse is specified.

  • git maintenance command

    A new builtin helps customize Git maintenance. Users can register their repositories to be maintained in the background using git maintenance start. Git v2.29.0, v2.30.0, v2.31.0

  • git sparse-checkout command

    A new builtin helps users manage their sparse-checkout files. A new "cone mode" improves performance for a common pattern type. Git v2.25.0

  • fetch.writeCommitGraph

    Write the Git commit-graph after every git fetch operation, ensuring that the repo is optimized for the new objects. Git v2.24.0

  • Feature macros

    Create new feature.* config options that group other config options together. Provides recommendations for users who do not want to read every config setting. Git v2.24.0

  • Incremental commit-graph format

    Speeds up git commit-graph write by amortizing writes across many small writes (and few big writes). Git v2.23.0

  • Multi-pack-index expire/repack

    Use the multi-pack-index to repack the object store incrementally in a highly-available environment. Git v2.23.0

  • "Sparse" push algorithm

    Speeds up git push for developers working in a small cone of a large repo. Git v2.21.0

  • Generation-number aware topological sort algorithm

    Significantly speeds up git log --graph calls when the commit-graph feature is enabled. Git v2.20.0

  • git multi-pack-index verify subcommand

    Verify the multi-pack-index file for corruption. Git v2.20.0

  • Rewrite commit walking code

    Collect and reorganize commit walking code, and improve several algorithms in the process. Git v2.20.0

  • Multi-pack-index

    Create a new file to index objects across multiple pack-files. Git v2.20.0

  • git commit-graph verify subcommand and gc.writeCommitGraph config option

    Verify the commit-graph file for corruption, and write the file automatically during repo maintenance. Git v2.20.0

  • Generation numbers

    Compute generation numbers in the commit-graph file and use them in some commit walks. Git v2.19.0

  • Commit-graph

    Create a new data structure and file format to store a compact representation of the commit history. Git v2.18.0

  • Speed up abbreviation algorithms

    Significantly improve the mechanism for computing the shortest unambiguous abbreviation of an object ID. Git v2.17.0
